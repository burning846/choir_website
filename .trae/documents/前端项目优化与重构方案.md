# 前端项目优化与重构方案

## 目标与范围
- 全面优化 React+TypeScript+Vite 前端项目的性能、可靠性、可维护性与可观测性
- 覆盖数据加载、错误处理、路由与 SEO、类型安全、构建与规范、测试与 CI

## 问题清单（审查结果）
- 数据重复请求：多组件独立 fetch 同一 JSON，缺少集中化数据层与缓存（示例： [Header.tsx](file:///Users/bytedance/Documents/personal/choir/src/components/Header.tsx#L10-L22)、[Hero.tsx](file:///Users/bytedance/Documents/personal/choir/src/components/Hero.tsx#L9-L19)、[Videos.tsx](file:///Users/bytedance/Documents/personal/choir/src/components/Videos.tsx#L28-L38)）
- 错误处理静默：普遍使用 .catch(() => {}) 无用户反馈与日志上报（如 [About.tsx](file:///Users/bytedance/Documents/personal/choir/src/components/About.tsx#L17-L18)、[Performances.tsx](file:///Users/bytedance/Documents/personal/choir/src/components/Performances.tsx#L13-L16)）
- 标题管理重复：document.title 在 [main.tsx](file:///Users/bytedance/Documents/personal/choir/src/main.tsx#L12-L20) 与 [Header.tsx](file:///Users/bytedance/Documents/personal/choir/src/components/Header.tsx#L18-L22) 均有设置
- 类型安全不足：TS 关闭 strict、noUnused*、skipLibCheck 等（[tsconfig.json](file:///Users/bytedance/Documents/personal/choir/tsconfig.json#L19-L24)）
- 未使用依赖：mammoth、fast-xml-parser、adm-zip、zustand 在 src 中无引用（[package.json](file:///Users/bytedance/Documents/personal/choir/package.json#L13-L43)）
- 路由简单：仅 "/" 与 "/other"，无 404 兜底与集中化 SEO（[App.tsx](file:///Users/bytedance/Documents/personal/choir/src/App.tsx#L8-L13)）
- 外部占位图合规风险：多个组件使用外部生成图服务（如 [Members.tsx](file:///Users/bytedance/Documents/personal/choir/src/components/Members.tsx#L38-L49)、[Conductor.tsx](file:///Users/bytedance/Documents/personal/choir/src/components/Conductor.tsx#L61-L66)）
- 构建调试受限：sourcemap 为 hidden（[vite.config.ts](file:///Users/bytedance/Documents/personal/choir/vite.config.ts#L7-L9)）
- ESLint 规则不足：缺少 a11y 与导入顺序等团队规范（[eslint.config.js](file:///Users/bytedance/Documents/personal/choir/eslint.config.js)）
- 缺少测试与 CI：无测试脚本与工作流

## 技术方案与实施步骤
1. 集中化数据层
- 新增 DocProvider（React Context）在顶层统一拉取与缓存中/英 JSON（[choir-doc.json](file:///Users/bytedance/Documents/personal/choir/public/choir-doc.json)、[choir-doc.en.json](file:///Users/bytedance/Documents/personal/choir/public/choir-doc.en.json)），暴露 useDoc/useLang Hooks
- 统一错误与加载态；实现请求去重与语言切换重载；移除各组件内重复 fetch
- 可选：后续引入 SWR/React Query 增强缓存与重试

2. 错误处理与观测
- 添加全局 ErrorBoundary；在数据层提供失败提示（toast/inline）与上报接口（Sentry 可插拔）
- 移除 .catch(() => {}) 静默，改为显式处理与上报

3. 标题与元信息管理
- 新增 useDocumentTitle Hook 或在 App 层集中设置标题；删除 main 与 Header 重复设置
- 引入 react-helmet-async（可选）集中管理 meta，补充基础 SEO 信息

4. TypeScript 严格化（渐进式）
- 开启 strict、noUnusedLocals、noUnusedParameters；保持 skipLibCheck 暂时开启以减少阻力
- 分模块修复类型与清理死码；对公共数据模型建立类型定义

5. 依赖清理与优化
- 移除未使用依赖：mammoth、fast-xml-parser、adm-zip、zustand；更新锁文件
- 评估打包体积变化并记录

6. 路由与页面体验
- 新增 404 Route 与基础兜底页；统一锚点导航策略，保留现有行为
- 为 “/other” 与首页补充 meta 与可访问性细节

7. 占位图资产化
- 在 public 增加本地占位图（logo/banner/avatar 等），统一引用，移除外部生成服务链接

8. 构建与调试
- 生产 sourcemap 调整为 true 或保留 hidden 并配套错误上报；记录取舍
- 保持 Vercel no-store 以避免过度缓存（[vercel.json](file:///Users/bytedance/Documents/personal/choir/vercel.json#L1-L15)）

9. 规范与质量保障
- 扩充 ESLint：jsx-a11y、import/order、complexity 等；集成 Prettier
- 引入 Vitest + React Testing Library：为 DocProvider、useDoc、关键组件添加 smoke 与行为测试
- 建立 GitHub Actions：安装依赖、lint、typecheck、test、构建

## 分阶段执行
- P0（用户体验优先）：集中化数据层、错误可见与上报、标题统一、404 页
- P1（工程质量）：TS 严格化基础、ESLint 规则、依赖清理
- P2（可观测与稳定）：sourcemap 策略、CI 工作流、基础单测
- P3（资产与SEO）：本地占位图、meta 集中与补全

## 验收标准
- 网络面板显示 doc JSON 每语言仅一次请求；切换语言无并发重复
- 加载失败有用户提示且日志可查看；document.title 管理唯一且正确
- 存在 404 页面；ESLint 与 typecheck 通过；基础单测通过并在 CI 中跑通

## 影响评估与兼容策略
- Context 引入对现有组件为内部重构，不改变对外 API；渐进式开启 TS 严格，避免一次性阻塞
- 占位图改为本地资源，减少外部依赖与合规风险；构建 sourcemap 方案与上报配套确保隐私与调试平衡
